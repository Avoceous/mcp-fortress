# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 0.1.x | ✅ |

## Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**

To report a security issue in MCP-Fortress:

1. Email: security@[yourdomain].com
2. Use GitHub's private vulnerability reporting:
   `https://github.com/Avoceous/mcp-fortress/security/advisories/new`

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

We will respond within **48 hours** and aim to release a patch within **7 days** for critical issues.

## Scope

In scope:
- Vulnerabilities in MCP-Fortress's own code that allow bypassing security controls
- False negatives: known attack patterns that MCP-Fortress fails to detect
- Dependency vulnerabilities

Out of scope:
- Vulnerabilities in upstream MCP servers (report to their maintainers)
- Issues requiring physical access to the deployment machine
- Social engineering attacks

## Hall of Fame

We credit all responsible disclosures in our release notes.
